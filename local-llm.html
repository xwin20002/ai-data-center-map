Methodology & Disclaimer

納入準則

  • OpenRouter 月用量 ≥ 10% 才納入本頁（Mistral / Phi / Gemma / Yi / Cohere 等 <10% 暫不收）
  • 2026 Q1 三家 anchor 合計 OpenRouter 用量 ~61%（Meta 23 + DeepSeek 22 + Qwen 16），剩 ~39% 為 long tail

衡量維度（4 軸）

  • 市佔：OpenRouter usage share + HuggingFace 月下載（兩個獨立 proxy 交叉驗證）
  • 最紅的模型：每年 hero model 列表（含 size + release date）
  • 模型大小：parameter range (smallest dense → largest MoE)
  • 趨勢：3 年 (2024-2026) OpenRouter share + HF download 曲線
  • (Bonus) Benchmark + Inference cost：MMLU 基準 vs $/1M token

關鍵 insights

  • DeepSeek 2026 Q1 inflection：用量 22% 接近 Meta 23%。HF 月下載 55M 已反超 Meta 45M，成 OSS 月下載 #1。R1 reasoning 開源 (Jan 2025) + V3.1 改 MIT license 是兩大轉折點。
  • Llama community license 限制：>700M MAU 用戶須申請額外授權；vs Qwen 3 Apache 2.0 + DeepSeek MIT 完全寬鬆，是 enterprise 採用 gate 的關鍵差異
  • Inference cost 5-30x 便宜：Together.ai Llama $0.6/M、DeepSeek $0.4/M、Qwen $0.5/M；對比 GPT-5 ~$2.5/M、Claude 4.6 ~$15/M
  • DeepSeek V3 訓練成本 $5.6M（vs OpenAI GPT-5 估 ~$1B+）— 引發業界重新評估 capex 結構，2025/01 NVIDIA 股價單日 -17% 反應
  • Qwen 3 size 範圍最廣：0.5B → 480B MoE（含 edge 到 frontier）；Qwen3-Coder-480B-A35B Apache 2.0 是 coding-specialized OSS 領先

License 比較

  • Meta Llama：Llama 3 community license（custom）— 商業可用但 >700M MAU 須申請
  • DeepSeek：V3.1 起 MIT — 業界最寬鬆，無商業/分發限制
  • Alibaba Qwen：Qwen 3 起 Apache 2.0 — enterprise 黃金標準（早期 Qwen 2.x 為 Qwen License 含 100M MAU 限）

產地

  • 🇺🇸 美國 1 家：Meta（NASDAQ:META，廣告 monetize 不靠 model 收費）
  • 🇨🇳 中國 2 家：DeepSeek（私有，量化基金 High-Flyer 全資）+ Alibaba（NYSE:BABA，Cloud + 開源 dual-axis）
  • 地緣政治觀察：OSS 三巨頭中國占兩家是 2024-2026 關鍵變化。中國 OSS 加速反應美對 NVIDIA 出口管制，激進開源是反封鎖戰術
  • 缺席的 vendor（後續可加）：Mistral 🇫🇷 (~5% 用量)、Microsoft Phi 🇺🇸 (~3%)、Google Gemma 🇺🇸 (~3%)、零一萬物 Yi 🇨🇳、TAIDE 🇹🇼、NVIDIA Nemotron 🇺🇸、Cohere 🇨🇦

與 Closed-Source LLM 對比

  • 本頁的 OSS 三巨頭 vs LLM Market 的 OpenAI / Anthropic / Google 形成完整 LLM 生態圖譜
  • Closed-source 重 ARR + MAU 商業 metrics；OSS 重 download + benchmark + license
  • Frontier 能力差距：MMLU 上 OSS 87-88 vs closed 88-92 (Claude 4.6 / GPT-5)，Gap 收窄至 2-5 點
  • 市場分流：closed 主導 consumer (ChatGPT 65%) 與大型 enterprise；OSS 主導 self-host / cost-sensitive / privacy-sensitive deployment
